# 3D Museum Platform

## Project Overview
Build a full-stack web-based 3D museum platform that allows users to browse artifacts, view interactive 3D models, and access curated information. Include an admin CMS for content management. The application should feature a dark, elegant, minimal design inspired by https://tszshanmuseum.org/en/.

---

## Design Requirements

### Visual Style & Theme
- **Dark theme** with sophisticated, museum-quality aesthetics
- Elegant serif fonts for headings, clean sans-serif for body text
- Minimalist layout with ample whitespace
- Subtle animations and transitions
- High contrast for accessibility
- Color palette: Deep blacks/dark grays as base, with gold/bronze accents for highlights
- Clean navigation with clear visual hierarchy

### Responsive Design
- Fully responsive across desktop, tablet, and mobile
- Mobile-first approach
- Touch-friendly controls for 3D viewer on mobile devices
- Optimized image loading for different screen sizes

---

## Technology Stack

### Frontend
- **Framework**: React with React Router for navigation
- **Styling**: TailwindCSS for responsive design
- **3D Viewer**: Three.js or Google Model Viewer for 3D model rendering
- **State Management**: React Context API or Zustand

### Backend
- **Runtime**: Node.js with Express.js
- **Authentication**: JWT-based authentication with bcrypt for password hashing
- **API Architecture**: RESTful API design
- **Input Validation**: Express-validator for sanitization
- **Rate Limiting**: Express-rate-limit for brute force protection

### Database
- **Primary Database**: PostgreSQL
- **ORM**: Sequelize for database operations
- **Schema Design**: Normalized structure with proper foreign keys

### File Storage
- **3D Models & Images**: AWS S3, Cloudflare R2, or DigitalOcean Spaces
- **CDN**: CloudFront or equivalent for fast global delivery
- **Image Optimization**: Sharp library for automatic image resizing

## 3. BACKEND API SPECIFICATIONS

### Authentication Endpoints

```
POST /api/auth/register
- Body: { email, password, confirmPassword, username }
- Returns: { success, message, user }

POST /api/auth/login
- Body: { email, password }
- Returns: { success, token, user }

POST /api/auth/logout
- Headers: { Authorization: Bearer <token> }
- Returns: { success, message }

POST /api/auth/forgot-password
- Body: { email }
- Returns: { success, message }

POST /api/auth/reset-password
- Body: { token, newPassword }
- Returns: { success, message }

GET /api/auth/me
- Headers: { Authorization: Bearer <token> }
- Returns: { user }
```

### Artifacts Endpoints

```
GET /api/artifacts
- Query params: ?page=1&limit=20&category=bronze&search=buddha&sortBy=newest
- Returns: { artifacts: [], total, page, totalPages }

GET /api/artifacts/:id
- Returns: { artifact }

POST /api/artifacts (Admin only)
- Headers: { Authorization: Bearer <admin-token> }
- Body: FormData with artifact details and files
- Returns: { success, artifact }

PUT /api/artifacts/:id (Admin only)
- Headers: { Authorization: Bearer <admin-token> }
- Body: Updated artifact data
- Returns: { success, artifact }

DELETE /api/artifacts/:id (Admin only)
- Headers: { Authorization: Bearer <admin-token> }
- Returns: { success, message }

GET /api/artifacts/:id/related
- Returns: { artifacts: [] }
```

### Categories Endpoints

```
GET /api/categories
- Returns: { categories: [] }

POST /api/categories (Admin only)
- Body: { name, description, thumbnail }
- Returns: { success, category }

PUT /api/categories/:id (Admin only)
- Body: { name, description, thumbnail }
- Returns: { success, category }

DELETE /api/categories/:id (Admin only)
- Returns: { success, message }
```

### Comments Endpoints

```
GET /api/comments/:artifactId
- Returns: { comments: [] }

POST /api/comments/:artifactId (Authenticated users)
- Headers: { Authorization: Bearer <token> }
- Body: { content }
- Returns: { success, comment }

PUT /api/comments/:id/approve (Admin only)
- Returns: { success, comment }

DELETE /api/comments/:id (Admin only)
- Returns: { success, message }
```

### Users Endpoints (Admin only)

```
GET /api/users
- Headers: { Authorization: Bearer <admin-token> }
- Returns: { users: [] }

GET /api/users/:id
- Returns: { user }

PUT /api/users/:id
- Body: { role, status }
- Returns: { success, user }

DELETE /api/users/:id
- Returns: { success, message }
```

### Admin Dashboard Endpoints

```
GET /api/admin/stats
- Headers: { Authorization: Bearer <admin-token> }
- Returns: { totalArtifacts, totalUsers, pendingComments, views }

GET /api/admin/analytics
- Returns: { mostViewed: [], recentActivity: [] }
```

## 4. DATABASE SCHEMA

### Users Table
```sql
id: UUID (Primary Key)
email: VARCHAR(255) UNIQUE NOT NULL
password: VARCHAR(255) NOT NULL (hashed)
username: VARCHAR(100)
role: ENUM('user', 'admin') DEFAULT 'user'
status: ENUM('active', 'blocked') DEFAULT 'active'
created_at: TIMESTAMP
updated_at: TIMESTAMP
```

### Artifacts Table
```sql
id: UUID (Primary Key)
title: VARCHAR(255) NOT NULL
short_description: TEXT
long_description: TEXT
category_id: UUID (Foreign Key -> Categories)
material: VARCHAR(100)
region: VARCHAR(100)
period: VARCHAR(100)
year: VARCHAR(50)
dimensions: JSONB
location: VARCHAR(255)
acquisition_date: DATE
has_3d_model: BOOLEAN DEFAULT FALSE
model_url: VARCHAR(500)
status: ENUM('draft', 'published') DEFAULT 'published'
views: INTEGER DEFAULT 0
created_by: UUID (Foreign Key -> Users)
created_at: TIMESTAMP
updated_at: TIMESTAMP
```

### Categories Table
```sql
id: UUID (Primary Key)
name: VARCHAR(100) UNIQUE NOT NULL
description: TEXT
thumbnail_url: VARCHAR(500)
parent_id: UUID (Foreign Key -> Categories, nullable)
order: INTEGER DEFAULT 0
status: ENUM('active', 'inactive') DEFAULT 'active'
created_at: TIMESTAMP
```

### Images Table
```sql
id: UUID (Primary Key)
artifact_id: UUID (Foreign Key -> Artifacts)
url: VARCHAR(500) NOT NULL
alt_text: VARCHAR(255)
is_primary: BOOLEAN DEFAULT FALSE
order: INTEGER DEFAULT 0
created_at: TIMESTAMP
```

### Comments Table
```sql
id: UUID (Primary Key)
artifact_id: UUID (Foreign Key -> Artifacts)
user_id: UUID (Foreign Key -> Users)
content: TEXT NOT NULL
status: ENUM('pending', 'approved', 'rejected') DEFAULT 'pending'
created_at: TIMESTAMP
updated_at: TIMESTAMP
```

### Tags Table
```sql
id: UUID (Primary Key)
name: VARCHAR(50) UNIQUE NOT NULL
created_at: TIMESTAMP
```

### Artifact_Tags Table (Junction)
```sql
artifact_id: UUID (Foreign Key -> Artifacts)
tag_id: UUID (Foreign Key -> Tags)
PRIMARY KEY (artifact_id, tag_id)
```

### Settings Table (Key-Value Store)
```sql
id: UUID (Primary Key)
key: VARCHAR(100) UNIQUE NOT NULL
value: TEXT
updated_at: TIMESTAMP
```

## 5. TECHNICAL IMPLEMENTATION DETAILS

### 3D Viewer Implementation

**Using Three.js:**
```javascript
// Basic setup
- Initialize scene, camera, renderer
- Add OrbitControls for interaction
- Load GLB/GLTF models using GLTFLoader
- Add lighting (ambient + directional)
- Handle window resize
- Implement auto-rotate functionality
- Add reset camera position button
```

**Or Using Model Viewer:**
```html
<!-- Simpler implementation -->
<model-viewer 
  src="model.glb"
  alt="Artifact"
  auto-rotate
  camera-controls
  shadow-intensity="1"
  ar
  ar-modes="webxr scene-viewer quick-look"
>
</model-viewer>
```

### File Upload Strategy
- Frontend: React Dropzone for drag & drop
- Backend: Multer middleware for handling multipart/form-data
- Validation: File type checking (GLB, GLTF for 3D; JPG, PNG, WEBP for images)
- Size limits: Images max 10MB, 3D models max 50MB
- Storage: Upload to S3/Spaces with unique file names (UUID + original extension)
- Response: Return CDN URL for immediate use

### Image Optimization Pipeline
- On upload, create multiple versions:
  - Thumbnail: 300x300px
  - Medium: 800x800px
  - Large: 1920x1920px
  - Original: Stored but not served directly
- Use Sharp library for processing
- Convert to WebP format for web delivery
- Lazy loading on frontend with blur-up placeholder

### Security Measures
- Helmet.js for security headers
- CORS configuration
- SQL injection prevention via parameterized queries
- XSS protection via input sanitization
- CSRF tokens for state-changing operations
- Rate limiting on all endpoints:
  - Auth endpoints: 5 requests per 15 minutes
  - General API: 100 requests per 15 minutes
- JWT expiration: 24 hours (refresh token: 7 days)
- Environment variables for sensitive data

### Error Handling
- Consistent error response format:
```json
{
  "success": false,
  "error": {
    "message": "User-friendly error message",
    "code": "ERROR_CODE",
    "details": {} // Optional developer info
  }
}
```
- Global error handler middleware
- Logging system (Winston or similar)
- User-friendly error messages on frontend
- Network error handling with retry mechanism

### Performance Optimization
- Implement caching:
  - Redis for frequently accessed data (optional)
  - HTTP caching headers for static assets
  - Browser caching strategy
- Database indexing on frequently queried fields
- Pagination for all list endpoints
- Lazy loading for images and 3D models
- Code splitting in React
- Compression middleware (gzip/brotli)
- CDN for all static assets

## 7. GETTING STARTED INSTRUCTIONS

### Environment Variables Setup
Create `.env` file in server directory:
```
NODE_ENV=development
PORT=5000
DATABASE_URL=postgresql://user:password@localhost:5432/museum_db
JWT_SECRET=your_jwt_secret_key_here
JWT_EXPIRES_IN=24h

# Storage (AWS S3 or DigitalOcean Spaces)
STORAGE_ENDPOINT=https://nyc3.digitaloceanspaces.com
STORAGE_BUCKET=museum-artifacts
STORAGE_ACCESS_KEY=your_access_key
STORAGE_SECRET_KEY=your_secret_key
CDN_URL=https://cdn.yourmuseum.com

# Email (for password reset)
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your_email@gmail.com
EMAIL_PASSWORD=your_app_password

# Admin Credentials (initial)
ADMIN_EMAIL=admin@museum.com
ADMIN_PASSWORD=SecurePassword123!
```

### Installation Steps
```bash
# Install dependencies
cd server && npm install
cd ../client && npm install

# Setup database
cd ../server
npx sequelize-cli db:create
npx sequelize-cli db:migrate
npx sequelize-cli db:seed:all

# Start development servers
# Terminal 1 - Backend
cd server && npm run dev

# Terminal 2 - Frontend
cd client && npm start
```
